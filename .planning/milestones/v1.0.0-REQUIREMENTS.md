# Requirements: chuhaihai

**Defined:** 2026-03-27
**Core Value:** 在不打断现有本地工作流的前提下，把飞书内容获取、筛选导入和相关配置集中到一个可持续维护的仓库里。

## v1 Requirements

### Feishu Bridge

- [x] **FSH-01**: 当前仓库包含可直接导入的 `feishu_screening_bridge` Python 包
- [x] **FSH-02**: 当前仓库保留从飞书文件链接或 `file_token` 下载内容的能力
- [x] **FSH-03**: 当前仓库保留从飞书 wiki/base URL 导出 bitable 内容的能力
- [x] **FSH-04**: 当前仓库保留下载 bitable 附件与同步任务上传视图的能力

### Config

- [x] **CFG-01**: `.env` 保留旧 `上传飞书` 所需的全部关键变量
- [x] **CFG-02**: `.env.example` 提供无敏感信息的等价配置模板
- [x] **CFG-03**: 迁移后的默认配置指向当前 sibling 项目结构，而不是旧绝对路径

### Testing

- [x] **TST-01**: 旧仓库的飞书桥接测试被一并迁入当前仓库
- [x] **TST-02**: 至少能在当前仓库执行一轮基础测试验证迁移未破坏导入

## v2 Requirements

### Consolidation

- **CNS-01**: 把 `抓取邮件` 中任务驱动邮件抓取所需的核心 `email_sync` 能力迁入当前仓库
- **CNS-02**: 把 `筛号` 的核心后端能力迁入当前仓库

## v3 Requirements

### Template Parsing

- **TMP-01**: 当前仓库包含可直接导入和执行的 workbook 模板解析模块
- **TMP-02**: 当前仓库支持在下载飞书需求模板后立即执行解析，并返回解析产物路径
- **TMP-03**: 当前仓库包含模板解析 standalone 行为与下载后解析桥接的测试验证

## v4 Requirements

### Screening Bridge

- **BRG-01**: 当前仓库支持把模板解析得到的 rulespec 与达人匹配名单直接写入 `筛号` 后端当前输入状态

## v5 Requirements

### Task-Driven Creator Source

- **FSH-05**: 当前仓库支持从任务上传记录中读取 `发信名单` 附件元数据，并按任务下载对应 workbook
- **BRG-02**: 当前仓库支持直接使用任务上传里的 `发信名单` workbook 生成 `data/<platform>/<platform>_upload_metadata.json`
- **BRG-03**: 当前仓库支持按任务名一次性完成“下载需求模板 + 下载发信名单 + 写入筛号输入状态”的任务驱动准备流程

## v6 Requirements

### Duplicate Match Adjudication

- **REV-01**: 当前仓库支持按 `last_mail_message_id`、缺失时回退 `last_mail_raw_path` 对 enrichment 输出做重复邮件分组
- **REV-02**: 当前仓库支持仅对重复邮件组执行 group-level LLM 裁决，并允许先跑小样本组而不是默认全量
- **REV-03**: 当前仓库支持输出 group-level 审核痕迹与 row-level 回填结果，保留人工复核所需字段

## v7 Requirements

### Production Duplicate Review Pipeline

- **REV-04**: 当前仓库支持从外部高置信匹配 workbook 生成生产复核所需的 `*_按我们去重.xlsx` 与 `*_按我们去重_去重.xlsx`，并补齐稳定的 `creator_dedupe_key` 与共享邮件辅助字段
- **REV-05**: 当前仓库支持输出 group-oriented 的 `*_llm_candidates.jsonl` 与 `*_llm_review.jsonl`，其中 LLM 决策口径为 `match_all / match_some / reject_group / uncertain`，选择键为 `matched_creator_dedupe_keys`
- **REV-06**: 当前仓库支持把 group-level LLM 审核结果回填成 `*_llm_reviewed.xlsx`，并产出最终 `*_llm_reviewed_keep.xlsx`
- **CFG-04**: duplicate review 的 LLM provider / model / wire API 切换只依赖 `.env` 中的 OpenAI-compatible 配置，不要求改代码

## v8 Requirements

### Keep-List Screening Integration

- **BRG-04**: 当前仓库支持直接使用 `*_llm_reviewed_keep.xlsx` 作为 `筛号` 主链的创作者输入源，并把分平台 metadata 写入 `data/<platform>/<platform>_upload_metadata.json`
- **SCR-01**: 当前仓库支持基于 keep-list 已写入的运行态触发 `Apify -> prescreen -> optional visual review -> export` 的本地运行闭环
- **SCR-02**: 当前仓库支持为 keep-list 下游运行输出机器可读 summary，明确记录 staged 数量、执行边界、视觉是否跳过和各平台导出产物路径

## Out of Scope

| Feature | Reason |
|---------|--------|
| 重做飞书 API 封装 | 当前迁移目标是保留已验证行为 |
| 替换现有本地导入链路 | 先保持兼容，后续再做统一重构 |
